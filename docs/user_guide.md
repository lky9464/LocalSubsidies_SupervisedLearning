# 지방보조금 부정수급 위험도 측정 — 사용자 가이드

로컬 PC 전용 웹 UI·파이프라인 안내입니다. 데이터와 점수는 외부로 전송되지 않습니다.

## 1. 시작하기

1. `configs/local.yaml` 에 `data_root` 경로를 설정합니다.
2. `git pull` 시 `web/out` 이 함께 오는지 확인합니다. 없거나 UI 소스를 수정했다면 `scripts/build_web.bat` 또는 Release의 `web-out.zip`을 준비합니다.
3. `RunWebNext.bat` 을 더블클릭합니다 (콘솔 창을 닫지 마세요).
4. 브라우저에서 `http://127.0.0.1:8600` 을 엽니다.

## 2. 메뉴 구성

| 메뉴 | 설명 |
|------|------|
| 대시보드 | **현재 Run 카드 선택** · 모델 평가(순위·타겟 포착) / 추론(4×4) |
| 데이터 등록 | 학습·평가 / 추론 CSV **누적 등록** 후, 사용할 파일만 **선택 저장** |
| ▼ 모델 학습 및 평가 | 학습 파이프라인 / 모델 비교·평가 / 타겟 포착 분포 (접이식) |
| ▼ 추론 | 추론 실행 / 결과 확인(점검 우선순위표·Excel) (접이식) |
| Run 이력 | 과거 Run 조회(현재 Run 선택은 대시보드) |
| 내 PC 사양 체크 | RAM/CPU/디스크 권장 여부 |
| 사용자 가이드 | 이 문서·PDF |
| 설정 | data_root·ops.sqlite 경로 |

## 3. 데이터 등록 · 선택

1. **학습·평가 raw** / **추론 raw** 각각에 CSV를 업로드합니다. 파일은 `{data_root}/raw` · `raw_inference`에 **누적**됩니다.
2. 목록에서 이번 작업에 쓸 파일만 체크한 뒤 **「선택 저장」** 합니다. (여러 개면 세로 결합)
3. 학습 실행·추론 실행 시 그 선택이 Run 설정에 **동결**되어 해당 Job에만 적용됩니다.
4. 다른 조합으로 돌리려면 삭제하지 말고 체크만 바꾼 뒤 다시 「선택 저장」하면 됩니다.
5. 동일 파일명 재업로드는 거부됩니다. 이름을 바꾸거나 기존 파일을 삭제하세요.

## 4. 파이프라인 순서

01 원본통합 → 02 타겟수정 → 03 전처리 → 04 누수점검 → 05 학습 →  
06 Feature중요도 → 07 평가·점수 → 08 모델순위 → 09 리포트 → 10 타겟 포착 분포  

추론(11)은 별도 메뉴 — **실행** / **결과 조회** 탭.  
산출: `scores/inference/{algo}_inference_scores.csv` · `_top.xlsx` (Test는 `scores/test/`).
점검 우선순위표 Excel: `{data_root}/algorithms/operations/ops_queue_inference.xlsx`.

## 5. 학습 옵션

- **분할**: 기간(time, `CRTR_YM`) 또는 랜덤(test_size 기본 0.3)
- **알고리즘**: 2개 이상 선택 필수 (주·보 4×4·비교 전제)
- **주·보 모델**: 고정(RF/CatBoost)이 아님 — **모델 비교·평가(08)** 와 대시보드에 표시된 `primary`/`aux` 확인
- **일괄 실행**: 시간 소요 경고 확인 후 백그라운드 Job으로 실행
- 다른 메뉴로 이동해도 Job은 계속되며, 상단 배너에서 진행률을 확인합니다.

## 6. 누수 FAIL 후 재개

04가 FAIL이면 의심·잔존 피처 목록이 표시됩니다.  
제외할 피처를 고른 뒤 **「제외 반영 후 03부터 재개」** 를 누르면 전처리·누수점검을 다시 실행합니다.

## 7. 타겟 포착 분포 vs 점검 우선순위표

주·보조 각각 상위% 구간: **A** 상위1% · **B** 1~5% · **C** 5~10% · **D** 10% 초과  
(주A~주D / 보A~보D, 우선순위 1~16 — 주등급 우선).

| 구간 | 메뉴 | 표시 |
|------|------|------|
| **Test(평가)** | 타겟 포착 분포 | **(A) 전체 4×4** + **(B) 실제 타겟 분포** — 포착 품질 확인 |
| **추론** | 추론 → 결과 확인 | **4×4 하나만** — 점검 대상 선정 |

상세: `docs/operations_criteria.md` §5.

## 8. 주의사항

- 브라우저 주소는 반드시 `127.0.0.1` 이어야 합니다.
- raw·점수 CSV·모델 파일은 GitHub에 올리지 마세요.
- 코드·UI 수정 후에는 `scripts/build_web.bat`(필요 시) 후 `RestartWeb.bat` / `RunWebNext.bat` 을 재시작하세요.
